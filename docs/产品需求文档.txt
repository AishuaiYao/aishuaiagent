# 「宝宝成长度量」小程序 - 产品需求文档 (PRD) v1.0
> 本文档专为 AI 编程 Agent（如 Cursor/Devin/GPT-Engineer）设计，所有描述均以"可编码实现"为目标。

---

## 1. 产品概述

### 1.1 核心价值
为0-6岁宝宝家长提供科学、趣味的**发展维度测评工具**，支持多宝宝档案管理与长期跟踪。

### 1.2 目标用户
0-6岁宝宝的父母/监护人。

### 1.3 产品类型
工具类小程序，强依赖微信登录与云存储。

### 1.4 导航模式
**汉堡菜单**（左上角图标），包含页面：
- 首页
- 全部测试
- 跟踪看板
- 我的宝宝
- 我的账号

### 1.5 UI 设计规范（全局约束）

> 以下规范适用于所有页面和组件，Agent 在生成样式代码时必须遵循。

| 设计维度 | 具体约束 |
|---------|---------|
| **设计哲学** | 轻量化软拟态 + 扁平化融合设计，温润通透、清新柔和，兼具现代移动端商务质感 |
| **圆角** | 所有卡片/按钮/弹窗统一使用 **20px~24px** 大圆角（`border-radius: 20px` 或 `24px`），输入框、标签使用 **12px** 小圆角 |
| **投影** | 极淡柔和阴影：`box-shadow: 0 4px 20px rgba(0,0,0,0.04)`，悬浮态加深至 `0 8px 30px rgba(0,0,0,0.08)`，无硬质边框（`border: none`） |
| **主色系** | 低饱和马卡龙渐变：主色 `#A8D8EA`（清冷蓝绿）、辅色 `#FFD3B4`（暖杏）、点缀 `#C7CEEA`（淡紫灰），所有渐变色差 < 30% |
| **背景色** | 清淡冷色：全局背景 `#F5F9FC`（冷白），卡片背景 `#FFFFFF` 或 `rgba(255,255,255,0.75)` 毛玻璃效果 |
| **图标风格** | 简约面性样式（非线框），统一使用 2px 圆角端点，尺寸 24px/32px/48px，颜色取自主色系 |
| **排版** | 留白充裕：页面左右边距 24px，上下模块间距 32px；字体层级：标题 18px/600，正文 15px/400，辅助 13px/400，颜色层级 `#1A2A3A` / `#4A5B6E` / `#8A9BAA` |
| **交互反馈** | 所有可点元素悬浮态有轻微上浮（`translateY(-2px)`）＋阴影加深，点击有 0.95 倍缩放反馈（`active:scale-95`） |
| **毛玻璃（可选）** | 顶部导航栏或浮层可使用 `backdrop-filter: blur(20px)` 配合半透明白色背景，增强通透感 |

## 2. 用户系统与数据模型（云开发集合）

### 集合1：`users`（用户表）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | string | 自动生成 |
| `openId` | string | 微信 openId（唯一） |
| `phone` | string | 授权手机号 |
| `nickName` | string | 微信昵称 |
| `avatarUrl` | string | 微信头像 |
| `createdAt` | date | 注册时间 |

### 集合2：`babies`（宝宝档案表）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | string | 自动生成 |
| `userId` | string | 关联 `users._id` |
| `name` | string | 宝宝昵称/小名 |
| `birthday` | date | 出生日期（用于计算月龄） |
| `gender` | string | 'male' 或 'female' 或 '' |
| `avatar` | string | 自定义头像（默认系统生成） |
| `note` | string | 备注（如早产等） |
| `isCurrent` | boolean | 是否为当前选中宝宝（每个用户仅一个为 true） |

### 集合3：`test_records`（测试记录表 - 结构化存储）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | string | 自动生成 |
| `userId` | string | 关联 `users._id` |
| `babyId` | string | 关联 `babies._id` |
| `testType` | string | 'attention' / 'attachment' / 'temperament' |
| `testName` | string | 如"注意力测试 v1.0" |
| `completedAt` | date | 完成时间 |
| `babyAgeMonths` | number | 测试时宝宝月龄（自动计算） |
| `scores` | object | 结构化得分，见下方各测试维度映射 |
| `dimensions` | object | 统一维度映射（用于雷达图），格式：<br>`{ "focus": 85, "emotion": 70, "social": 60, "sensory": 90 }`<br>取值范围 0-100 |
| `summary` | string | 文字结论（如"安全型依恋"） |
| `suggestions` | array | 建议列表，如 `["多玩躲猫猫", "增加回应性护理"]` |
| `isShared` | boolean | 是否已生成分享卡片 |

#### 各测试 `scores` 子字段示例：
- **attention**: `{ "sustained": 18, "selective": 20, "shifting": 15 }` （满分各25）
- **attachment**: `{ "security": 85, "avoidance": 20, "resistance": 15 }` （满分100）
- **temperament**: `{ "activity": 30, "rhythmicity": 25, "approach": 40, "adaptability": 35, "intensity": 20, "mood": 45, "persistence": 30, "distractibility": 25, "threshold": 20 }` （满分各50）

## 3. 页面清单与跳转逻辑

| 页面名称 | 路径 | 功能简述 |
|---------|------|---------|
| 首页 | `pages/index/index` | 展示当前宝宝、快捷测试入口、最近测试记录 |
| 全部测试 | `pages/tests/tests` | 列出所有测试项（含状态：可测/即将上线） |
| 测试答题 | `pages/test/quiz` | 逐题作答（量表类） |
| 测试结果 | `pages/test/result` | 展示得分、维度、建议、历史对比 |
| 跟踪看板 | `pages/dashboard/dashboard` | 时间线 + 雷达图 |
| 我的宝宝 | `pages/babies/babies` | 宝宝列表、切换当前宝宝、添加/编辑宝宝 |
| 我的账号 | `pages/account/account` | 显示用户信息，退出登录 |

**跳转关系**：
- 首页 →（点击测试卡片）→ 测试答题 → 测试结果 →（返回）→ 首页/看板
- 汉堡菜单 → 任意页面
- 测试结果 →（点击"查看跟踪"）→ 跟踪看板（自动筛选该宝宝）

## 4. 各页面 UI 布局（块级描述）

### 4.1 首页 (`index`)
- **顶部区域**（150px高）：当前宝宝头像（圆形，60px）+ 昵称 + 月龄标签（如"24个月"），右侧汉堡菜单按钮。
- **欢迎语**：如"今天想测测宝宝的什么呢？✨"
- **快捷入口区**（横向滚动卡片）：
  - 卡片1：注意力测试（带小图标）
  - 卡片2：亲子依恋测试
  - 卡片3：气质类型测试
  - 卡片4：更多测试（跳转至全部测试）
- **最近记录区**（列表）：显示最近3条测试记录（测试名 + 日期 + 一句话结论），点击可查看结果详情。

### 4.2 全部测试 (`tests`)
- **列表形式**，每条包含：
  - 测试图标（左）
  - 测试名称 + 简短描述（如"通过日常行为评估宝宝注意力水平"）
  - 状态标签：`可测`（绿色）/ `即将上线`（灰色）
  - 点击可测项 → 进入答题页
- 三个量表项均标为 `可测`；交互式实验和图文指导标为 `即将上线`（不可点击）。

### 4.3 测试答题 (`quiz`)
- **顶部进度条**（0% ~ 100%）
- **题号**（如"第 3 / 15 题"）
- **题干**（大号字体，居中）
- **选项**（单选按钮样式，大点击区域）：
  - 采用 Likert 或单选选项（如"从不/偶尔/经常/总是"或"是/否"）
- **底部"下一题"按钮**（仅在选中选项后可用）
- **最后一题按钮变为"提交答案"**，点击后计算得分并跳转结果页。
- **重要**：无自动保存进度；退出即清空当前答题状态。

### 4.4 测试结果 (`result`)
- **顶部**：测试名称 + 完成日期 + 宝宝月龄
- **总评分/等级**（大号数字 + 徽章，如"82分 · 良好"）
- **维度细分**（柱状图或条形图，使用 Canvas 或 echarts-for-weixin）
- **文字解读/建议区**（卡片样式，带小标题"解读与建议"）
- **历史对比**（如果存在前次同类型测试）：显示"较上次 +5 分"或"持平"
- **底部按钮**：
  - "查看跟踪看板"（跳转 dashboard）
  - "保存/分享卡片"（生成图片，保存到相册）
  - "再测一次"（回到该测试答题页，重置进度）

### 4.5 跟踪看板 (`dashboard`)
- **顶部**：当前宝宝头像 + 名称 + 月龄
- **时间线**（按时间倒序显示所有测试记录卡片，每个卡片显示测试名 + 结论 + 日期）
- **雷达图**（使用 echarts 雷达图），展示当前宝宝在所有统一维度（focus/emotion/social/sensory）上的**最新一次各测试的均分**或**最近一次综合得分**。
- **维度筛选器**（可点击维度高亮显示）
- 如果无数据，显示空态插画 + "还没有测试记录，去测一测吧"。

### 4.6 我的宝宝 (`babies`)
- **顶部**："我的宝宝们" 标题 + "添加宝宝"按钮
- **宝宝卡片列表**（每个卡片）：
  - 头像 + 昵称 + 月龄 + 性别图标
  - 若为当前选中宝宝，显示"当前"标签（绿色小圆点）
  - 卡片右侧有"切换"按钮（将该宝宝设为当前）
  - 卡片底部有"编辑"和"删除"按钮（删除需二次确认）
- **添加宝宝表单**（弹窗或新页面）：昵称、出生日期（picker）、性别、头像（从系统预设头像选择）、备注（可选）。

### 4.7 我的账号 (`account`)
- 头像 + 昵称 + 手机号（脱敏显示）
- "退出登录"按钮（清除本地缓存，但不删除云端数据）

## 5. 交互状态机（通用）
- **加载态**：页面顶部显示骨架屏或 loading 动画（使用微信原生 `wx.showLoading`）
- **空态**：无数据时显示插画 + 引导文字 + 行动按钮（如"去测试"）
- **错误态**：网络错误时显示重试按钮（`wx.showToast` 提示）
- **所有按钮点击**：必须有触感反馈（`hover-class` 或 `bindtap` 配合震动反馈可选）

## 6. 技术实现提示（给 Agent 的备注）
- **框架**：原生微信小程序 或 Taro（建议 Taro 3 + React/Vue，便于组件复用）
- **UI 组件库**：建议使用 `TDesign Mini` 或 `Vant Weapp`，以快速实现卡片、按钮、弹窗、进度条。
- **图表**：使用 `echarts-for-weixin` 绘制雷达图和柱状图。
- **云开发**：使用 `wx.cloud.database()` 操作集合，`wx.cloud.callFunction` 用于复杂统计（如雷达图聚合计算）。
- **状态管理**：建议使用 `mobx-miniprogram` 或 `pinia`（Taro）管理全局当前宝宝、用户信息。
- **图片资源**：所有插画使用 SVG 或 PNG，存放于 `assets/images`，风格统一为暖色毛绒质感。
- **分享卡片生成**：使用 `wx.canvasToTempFilePath` 绘制结果卡片。

## 8. 非功能性需求
- 首屏加载时间 < 2 秒（开启云开发缓存）
- 所有用户数据按 `userId` 和 `babyId` 建立索引，保证查询效率
- 测试过程无闪退，答题选项状态实时保存于局部变量（非持久化）
